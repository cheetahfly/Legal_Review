📱 真机测试指南 Legal_Review

从零开始在真实 Android 手机上测试本 App — 面向新手，每一步都告诉你点哪里。
⏱ 所需时间：首次约 15-30 分钟（含开发者选项配置），后续约 5 分钟。
📦 需要：一台 Android 手机 + USB 数据线（要能传数据）。
目录 0. 准备工作 手机端：开启开发者选项 电脑端：确认 adb 可用 1. 连接手机 2. 编译并安装 APK 3. 启动 App 4. 跑自动化 instrumentation 测试 5. 按手工清单走一遍 6. 看日志（排查问题用） 7. 常见问题 快速对照表

0 准备工作

📱 手机端：开启「开发者选项」和「USB 调试」

这是最关键的一步，每个手机差不多但菜单位置可能略有不同：

打开手机 「设置」 → 「关于手机」
找到 「版本号」（有的叫「Build Number」），连续点击 7 次
底部会弹出提示「您已进入开发者模式」或类似文字
返回设置首页，在搜索框搜 「开发者选项」
进入后打开 「USB 调试」 开关 ✅
💡 品牌差异：
小米/Redmi：还需要在开发者选项里打开 「USB 安装」 和 「USB 调试（安全设置）」。
华为/荣耀：可能需要输一次锁屏密码来确认开启。
三星：版本号在「关于手机」→「软件信息」里。
OPPO/vivo：版本号通常在「关于手机」→「版本信息」里。

💻 电脑端：确认 adb 命令可用

打开终端（Git Bash），跑：

export JAVA_HOME="C:/Users/user/.jdks/jdk-17/jdk-17.0.11+9"
export PATH="$JAVA_HOME/bin:$PATH"
export PATH="/f/Android/Sdk/platform-tools:$PATH"

# 验证 adb 版本
adb --version

正常会显示版本号（如 Android Debug Bridge version 1.0.41）。如果报 adb: command not found，确认上面的路径对不对。

💡 每次都要打 export？ 是的，Git Bash 每次新开窗口都要设一次。
一劳永逸的办法：把三行 export 加到 ~/.bashrc： 
echo 'export JAVA_HOME="C:/Users/user/.jdks/jdk-17/jdk-17.0.11+9"' >> ~/.bashrc
echo 'export PATH="$JAVA_HOME/bin:\$PATH"' >> ~/.bashrc
echo 'export PATH="/f/Android/Sdk/platform-tools:\$PATH"' >> ~/.bashrc

1 连接手机

用 USB 数据线把手机插到电脑
手机下拉通知栏 → 把「USB 用途」改成 「传输文件」（MTP），不是「仅充电」
手机会弹框「允许 USB 调试吗？」，勾选「一律允许」，点确定
没弹框？ 重新拔插 USB 线。还是没弹？去开发者选项里关掉再打开 USB 调试，再插线。

然后跑验证命令：

adb devices -l

成功时的样子：

List of devices attached
XXXXXXXXXXXX    device product:xxx model:XXX_XX transport_id:X

失败时的样子：

List of devices attached
（空 — 电脑完全没发现手机）
List of devices attached
XXXXXXXXXXXX    unauthorized
（手机连上了，但没确认授权）
常见坑：
  • 换一根线！很多 USB 线只能充电，不能传数据。换手机原装线或标称「数据线」的。
  • 电脑 USB 口不行？ 插机箱背面，别插机箱前面板。
  • Windows 驱动没装？ 连上后 Windows 应该自动装驱动，如果设备管理器有黄叹号，去手机官网下 USB 驱动手动装。

2 编译并安装 APK

先切到项目目录，编译 debug APK：

cd F:/my_git/Legal_Review
./gradlew :app:assembleDebug

然后装到手机上：

# 首次安装
adb install -r app/build/outputs/apk/debug/app-debug.apk

# 如果已经装过，想完全重来（清数据）：
adb uninstall com.legalreview
adb install app/build/outputs/apk/debug/app-debug.apk

安装成功会显示：

Performing Streamed Install
Success

3 启动 App

adb shell am start -n com.legalreview/.MainActivity

手机上应该弹出主界面，标题显示「协议把关」。如果没弹，看日志排查。

4 跑自动化 instrumentation 测试

这一步不是必须的，但能自动验证更多场景（设置加密、分析流水线在真机上的表现）。如果你时间充裕，跑一遍会多一层保障。 
# 确保手机还在线
adb devices -l

# 跑所有 instrumentation 测试（3 个类、6 个用例）
./gradlew :app:connectedDebugAndroidTest

# 看 HTML 报告
start app/build/reports/androidTests/connected/index.html

每个测试应该显示绿色 ✅。如果有红色的失败，截图给我。

5 按手工清单走一遍

详细文档在 docs/e2e-checklist.md。下面是最快的验证路径：

⏱ 10 分钟快速路径（覆盖 80% 功能）

手机点 App 图标进入主界面 → 看到「协议把关」
点 「去开启悬浮窗权限」 → 跳转到系统设置 → 打开悬浮窗 → 返回 App
下拉选择模型（保持 DeepSeek 即可）→ 填入 API Key → 点 「保存设置」
点 「授权并启动悬浮按钮」 → 允许屏幕共享
按 Home 键回到桌面 → 看到 悬浮按钮
打开浏览器 → 进去一个有协议条款的页面（比如搜"用户协议模板"）
点悬浮按钮 → 等几秒 → 弹出风险条款结果
验证结果包含至少 1 条风险项（比如「自动续费」）

6 看日志（排查问题用）

只显示我们 App 的输出，不刷系统垃圾：

# 清空旧日志
adb logcat -c

# 过滤只显示我们的 TAG
adb logcat LegalOverlayService:V ScreenCaptureManager:V OcrRecognizer:V \
           AnalysisOrchestrator:V OpenAiCompatibleLlmClient:V \
           SettingsRepository:V *:S

常见日志关键词：

TAG看到什么表示正常
LegalOverlayServiceonClick → 截图流程开始
ScreenCaptureManagerMediaProjection started → 截屏就绪
OcrRecognizerrecognize started → 文本识别开始
AnalysisOrchestratorlocal findings=N → 本地规则命中 N 条
AnalysisOrchestratorllm call took=Xms → LLM 分析耗时
OpenAiCompatibleLlmClientHTTP 401 → API Key 不对（但仍能出本地结果）

7 常见问题

症状怎么办
adb devices 啥也没有 换 USB 线；改 USB 模式为「传输文件」；换电脑 USB 口
unauthorized 手机解锁，看有没有弹授权框
装不上 INSTALL_FAILED_UPDATE_INCOMPATIBLE adb uninstall com.legalreview 再装
悬浮按钮没出现 去系统设置 → 应用 → 协议把关 → 显示悬浮窗，确认已开启
点悬浮按钮没反应 adb logcat LegalOverlayService:V *:S 看日志
分析结果全是空的 检查 API Key 是否正确；看日志有没有 OCR 输出
打开 App 闪退 adb logcat *:E 看崩溃堆栈，贴给我
